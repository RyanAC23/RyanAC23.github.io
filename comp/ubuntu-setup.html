# New Ubuntu Mate Setup ######################################################
##############################################################################
Set up partitions. This includes a swap space, doc space, and / root. This will
need its own section to quickly be able to set up the dual boot of Mate - Arch
with proper swap sharing.
Installation is easy. Once it's installed, here's what I like to use.

### terminal emulator ###
- terminator # terminal emulator based on GNOME terminal. Works well.
- urxvt # an x window virtual terminal. Supports w3m for images, so I might move here.
    - sudo apt install rxvt-unicode
    - w3m # You'll need this if you want to set up urxvt to display images, e.g. with ranger.
    - libx11-dev & libxft-dev # you may need to apt install these to make urxvt pretty.
    - sudo apt install xsel # urxvt doesn't come with copy/paste out of the box.
    - Make sure you have your .Xresources from bitbucket, and also perl extensions in .urxvt/ext/
        - If using i3, there are some extra steps necessary to make transparency work....

### Dotfiles ###
cp -rf .vim  ~/
> cp -rt .config ~/.config # merges the two config folders.
    # /.config
    #     |
    # /asunder /neofetch /transmission /pluma /terminator
> cp .tmux.conf ~/.tmux.conf

- .bashrc:
append this to the bottom of the file:
    #######################
    # Extra Customization #
    #######################
    # enable terminal vi mode
    set -o vi
    # enable optional shell feature autocd
    shopt -s autocd

    # autostart multiplexer
    if command -v tmux &> /dev/null && [ -z "$TMUX" ]; then
            tmux attach -t default || tmux new -s default
    fi

    if [ -d "~/bin" ] ; then
    export PATH="$~/bin:$PATH"
    fi

    # Add conda env quickstart command
    source ~/.config/.environment_site


### Settings ###
- I had to enable line-out audio on my desktop. This may not be an issue elsewhere.
- Screen tearing with NVIDIA graphics card:
    - Open Nvidia X Server Settings > X Server Display Configuration > Advanced >
    Force Full Composition Pipeline > Save to X Configuration File >
        '/etc/X11/xorg.conf'; or
- nvidia-settings --assign CurrentMetaMode="nvidia-auto-select
    +0+0 { ForceFullCompositionPipeline = On }"
- Mate Tweak > unclick 'show desktop icons'.
- set terminator as the preferred terminal

### Python / Jupyter ###
> mkdir ~/data/apps
> cd ~/data/apps
> wget https://repo.continuum.io/miniconda/Miniconda3-latest-Linux-x86_64.sh
> bash Miniconda3-latest-Linux-x86_64.sh
    - install in ~/data/apps/miniconda
# install minimal miniconda in base env, then make new environments for projects
# as necessary.
> conda update -y conda
> conda update -y --all
> conda list
# Now make new environments with ipykernel, including a base jupyter environment.
# The jupyter environment MUST include nb_conda, and all others NEED ipykernel.
> wget https://anaconda.org/mforbes/jupyter/2019.03.31.153852/download/environment.jupyter.yml
> conda env create --file environment.jupyter.yml
> conda create -n py3
> conda activate py3
> conda install ipykernel numpy matplotlib scipy
> conda deactivate
# This should now give you a jupyter server environment with py3 as a kernel option.
    - At the end of any massive install session:
    > conda update -y conda
    > conda update -y --all
    > conda clean -p --dry-run
    > conda clean -p -y
